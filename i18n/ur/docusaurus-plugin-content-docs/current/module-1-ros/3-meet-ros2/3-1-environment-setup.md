# سبق 3.1: ماحول کا سیٹ اپ

ROS 2 سسٹم کے ساتھ تعامل کرنے سے پہلے، آپ کو اس تک رسائی کی ضرورت ہے۔ اس سبق میں، آپ اپنا ماحول تیار کریں گے اور چلائیں گے۔

پچھلے باب سے آپ کے ہارڈ ویئر ٹائر کی بنیاد پر، آپ کے پاس دو بنیادی اختیارات ہیں۔

---

### آپشن A: ٹائر 1 (کلاؤڈ پر مبنی)

ٹائر 1 کے طلباء کے لیے، ہم کلاؤڈ پر مبنی روبوٹکس پلیٹ فارم استعمال کرنے کی تجویز کرتے ہیں۔ یہ شروع کرنے کا سب سے تیز اور آسان طریقہ ہے، کیونکہ اس میں مقامی انسٹالیشن کی ضرورت نہیں ہے۔

**سفارش: دی کنسٹرکٹ**

1.  **سائن اپ کریں:** [دی کنسٹرکٹ](https://www.theconstructsim.com/) پر جائیں اور ایک مفت اکاؤنٹ کے لیے سائن اپ کریں۔
2.  **ایک ROSject لانچ کریں:** ROS 2 Humble کورس تلاش کریں یا ایک نیا ROSject بنائیں۔ یہ آپ کے براؤزر میں ایک ورچوئل مشین شروع کرے گا جس میں ROS 2، ایک کوڈ ایڈیٹر، اور ایک گرافیکل ڈیسک ٹاپ سب پہلے سے ترتیب شدہ ہیں۔
3.  **ایک ٹرمینل کھولیں:** مربوط ڈیسک ٹاپ میں ایک ٹرمینل ایپلی کیشن ہوگی۔ یہ وہ جگہ ہے جہاں آپ اس باب کے لیے تمام os2 کمانڈز چلائیں گے۔
4.  **ماحول کو سورس کریں:** ROS میں، آپ کو ہر نئے ٹرمینل میں ROS کمانڈز کو دستیاب کرنے کے لیے ایک سیٹ اپ فائل کو "سورس" کرنا ہوگا۔ عین مطابق کمانڈ پلیٹ فارم کے لحاظ سے مختلف ہو سکتی ہے، لیکن یہ کچھ اس طرح نظر آئے گی:
    `ash
    source /opt/ros/humble/setup.bash
    `
    آپ کا کلاؤڈ ماحول شاید یہ آپ کے لیے خود بخود کر دے۔

**فوائد:**
*   آپ کی مقامی مشین پر صفر انسٹالیشن۔
*   معیاری ماحول اس بات کو یقینی بناتا ہے کہ تمام مثالیں باکس سے باہر کام کریں گی۔

---

### آپشن B: ٹائر 2+ (مقامی انسٹالیشن)

ایک وقف شدہ لینکس مشین یا ڈوئل بوٹ سیٹ اپ والے طلباء کے لیے، ROS 2 کو مقامی طور پر انسٹال کرنا طویل مدتی بہترین آپشن ہے۔

**سرکاری انسٹالیشن گائیڈ**

سرکاری ROS 2 دستاویزات انسٹالیشن کی ہدایات کے لیے واحد سچائی کا ذریعہ ہیں۔ اپنے مخصوص آپریٹنگ سسٹم کے لیے گائیڈ پر عمل کریں۔

**ہدف کی تقسیم: ROS 2 Humble Hawksbill**

*   **ROS 2 Humble انسٹالیشن گائیڈ پر جائیں:** [docs.ros.org](https://docs.ros.org/en/humble/Installation.html)
*   **اپنا OS منتخب کریں:** بنیادی تعاون یافتہ OS **Ubuntu 22.04 (Jammy Jellyfish)** ہے۔
*   **Debian ہدایات پر عمل کریں:** یہ آپ کو ROS 2 ریپوزٹریز کو شامل کرنے اور os-humble-desktop پیکیج کو انسٹال کرنے میں رہنمائی کرے گا۔

**کلیدی اقدامات:**
1.  **لوکیل اور ذرائع متعین کریں:** یقینی بنائیں کہ آپ کا سسٹم packages.ros.org سے پیکیجز قبول کرنے کے لیے ترتیب دیا گیا ہے۔
2.  **ROS 2 پیکیجز انسٹال کریں:** مرکزی کمانڈ یہ ہوگی:
    `ash
    sudo apt install ros-humble-desktop
    `
3.  **ماحول کو سورس کریں:** بالکل کلاؤڈ کی طرح، آپ کو ہر ٹرمینل میں سیٹ اپ فائل کو سورس کرنا ہوگا۔ اس لائن کو اپنی ~/.bashrc فائل کے آخر میں شامل کریں تاکہ یہ ہر نئے ٹرمینل کے لیے خود بخود چل جائے:
    `ash
    echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
    `
    پھر، اپنے موجودہ ٹرمینل میں source ~/.bashrc چلائیں۔

**توثیق**

چاہے آپ نے کوئی بھی آپشن منتخب کیا ہو، آپ ایک سادہ کمانڈ چلا کر اپنی انسٹالیشن کی توثیق کر سکتے ہیں:
`ash
ros2 --help
`
اگر آپ کا ماحول صحیح طریقے سے ترتیب دیا گیا ہے، تو یہ دستیاب os2 کمانڈز کی ایک فہرست پرنٹ کرے گا۔ اگر آپ کو "command not found" کی خرابی ملتی ہے، تو اس کا مطلب ہے کہ آپ نے سیٹ اپ فائل کو صحیح طریقے سے سورس نہیں کیا ہے۔
