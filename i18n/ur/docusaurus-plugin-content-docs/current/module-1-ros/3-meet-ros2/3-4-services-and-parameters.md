# سبق 3.4: خدمات اور پیرامیٹرز

ٹاپکس مسلسل ڈیٹا اسٹریمز کے لیے بہترین ہیں، لیکن کبھی کبھی آپ کو درخواست/جواب کے تعامل کی ضرورت ہوتی ہے۔ مثال کے طور پر، "موجودہ وقت کیا ہے؟" یا "براہ کرم سمیولیشن کو دوبارہ ترتیب دیں۔" اس کے لیے **خدمات** ہیں۔

**پیرامیٹرز**، دوسری طرف، ایک نوڈ کو دوبارہ شروع کیے بغیر رن ٹائم پر ترتیب دینے کا ایک طریقہ ہیں۔

آئیے ان تصورات کو Turtlesim کا استعمال کرتے ہوئے دریافت کریں۔ اپنے 	urtlesim_node کو چلاتے رہیں۔

---

## خدمات: درخواست/جواب ماڈل

ایک خدمت میں ایک **سرور** (جو خدمت فراہم کرتا ہے) اور ایک **کلائنٹ** (جو خدمت کو کال کرتا ہے) ہوتا ہے۔ یہ ایک دو طرفہ گفتگو ہے۔

آئیے دیکھتے ہیں کہ 	urtlesim_node کون سی خدمات فراہم کرتا ہے۔ ایک نئے ٹرمینل میں، چلائیں:
`ash
ros2 service list
`
آپ کو ایک لمبی فہرست نظر آئے گی، بشمول:
`
/clear
/kill
/reset
/spawn
/turtle1/set_pen
...
`
یہ وہ تمام اعمال ہیں جن کی آپ 	urtlesim_node سے "درخواست" کر سکتے ہیں۔

آئیے /clear سروس کو کال کرنے کی کوشش کریں۔ یہ خدمت کچھوے کے چھوڑے ہوئے راستے کو مٹاتی ہے۔ کمانڈ لائن سے کسی خدمت کو کال کرنے کے لیے، ہم os2 service call استعمال کرتے ہیں۔
`ash
ros2 service call /clear std_srvs/srv/Empty
`
*   /clear: خدمت کا نام۔
*   std_srvs/srv/Empty: خدمت کی **قسم**۔ اس صورت میں، یہ ایک "خالی" خدمت ہے، جس کا مطلب ہے کہ درخواست میں کوئی ڈیٹا شامل کرنے کی ضرورت نہیں ہے۔

جب آپ اسے چلاتے ہیں، تو آپ دیکھیں گے کہ کچھوے کا راستہ غائب ہو جاتا ہے۔

### دلائل کے ساتھ خدمات

کچھ خدمات کو دلائل کی ضرورت ہوتی ہے۔ آئیے ایک نیا کچھوا پیدا کریں۔ پہلے، آئیے دیکھتے ہیں کہ /spawn سروس کو os2 service info کے ساتھ کون سی معلومات درکار ہیں۔
`ash
ros2 service info /spawn
`
یہ آپ کو بتائے گا کہ خدمت کی قسم 	urtlesim/srv/Spawn ہے۔ ہمیں مزید تفصیلات کی ضرورت ہے۔
`ash
ros2 interface show turtlesim/srv/Spawn
`
یہ درخواست اور جواب کی ساخت کو ظاہر کرتا ہے۔
`
float32 x
float32 y
float32 theta
string name # Optional. A unique name will be created if this is empty.
---
string name
`
درخواست (اوپر ---) کو ایک x, y, تھیٹا، اور ایک اختیاری نام کی ضرورت ہے۔ جواب (نیچے ---) نئے کچھوے کا نام واپس کرے گا۔

اب آئیے اسے کال کریں:
`ash
ros2 service call /spawn turtlesim/srv/Spawn "{x: 1.0, y: 1.0, theta: 0.5, name: 'turtle2'}"
`
ایک نیا کچھوا 	urtle2 نامی پوزیشن (1, 1) پر سمیولیشن میں ظاہر ہوگا!

## پیرامیٹرز: ایک نوڈ کو ترتیب دینا

پیرامیٹرز ہمیں ایک نوڈ کی ترتیب کو چلاتے ہوئے تبدیل کرنے کی اجازت دیتے ہیں۔

آئیے دیکھتے ہیں کہ 	urtlesim_node کے پاس کون سے پیرامیٹرز ہیں۔
`ash
ros2 param list
`
آپ کو نظر آنا چاہیے:
`
/turtlesim:
  background_b
  background_g
  background_r
  use_sim_time
`
/turtlesim نوڈ کے پاس سمیولیشن کے پس منظر کے رنگ کو کنٹرول کرنے کے لیے پیرامیٹرز ہیں۔

آئیے سرخ جزو کی موجودہ قدر پڑھتے ہیں۔
`ash
ros2 param get /turtlesim background_r
`
یہ 69 واپس کرے گا۔

اب، آئیے اسے تبدیل کریں۔ ہم os2 param set کا استعمال کرکے پس منظر کو ایک اچھے پیلے رنگ میں تبدیل کر سکتے ہیں (سرخ=255، سبز=255، نیلا=0)۔
`ash
ros2 param set /turtlesim background_r 255
ros2 param set /turtlesim background_g 255
ros2 param set /turtlesim background_b 0
`
جیسے ہی آپ ان کمانڈز کو چلاتے ہیں، آپ دیکھیں گے کہ Turtlesim ونڈو کا پس منظر کا رنگ حقیقی وقت میں تبدیل ہوتا ہے۔ آپ نے ابھی ایک چلتے ہوئے ROS 2 نوڈ کو دوبارہ شروع کیے بغیر دوبارہ ترتیب دیا ہے۔

یہ روبوٹس کی ٹیوننگ اور کیلیبریٹنگ کے لیے ایک انتہائی طاقتور خصوصیت ہے۔ آپ موٹر کنٹرولر گینز، سینسر فلٹر سیٹنگز، اور بہت کچھ ایڈجسٹ کر سکتے ہیں، جبکہ سسٹم لائیو ہے۔
