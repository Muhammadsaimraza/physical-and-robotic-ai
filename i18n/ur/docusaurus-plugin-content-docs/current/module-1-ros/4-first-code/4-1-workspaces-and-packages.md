# سبق 4.1: ورک اسپیسز اور پیکیجز

کوڈ لکھنے سے پہلے، ہمیں اپنی فائلوں کو منظم کرنے کی ضرورت ہے۔ ROS 2 میں ترقی کے لیے ایک معیاری، روایتی ڈھانچہ ہے جسے **ورک اسپیس** کہا جاتا ہے۔

## ROS 2 ورک اسپیس

ورک اسپیس ایک ڈائرکٹری ہے جس میں آپ کے ROS 2 پیکیجز ہوتے ہیں۔ معیاری ڈھانچہ اس طرح نظر آتا ہے:
`
ros2_ws/
├── src/
├── build/
├── install/
└── log/
`
*   src/: یہ **سورس** ڈائرکٹری ہے۔ یہ واحد ہے جسے آپ براہ راست چھوئیں گے۔ آپ اپنے تمام پیکیجز اس فولڈر کے اندر رکھیں گے۔
*   uild/: یہ ایک عارضی ڈائرکٹری ہے جہاں ROS 2 کا بلڈ ٹول، colcon، آپ کے کوڈ کو مرتب کرتے وقت درمیانی فائلیں ڈالے گا۔ آپ کو اس ڈائرکٹری میں کبھی بھی ترمیم نہیں کرنی چاہیے۔
*   install/: ایک کامیاب تعمیر کے بعد، colcon حتمی، قابل نصب فائلیں یہاں رکھے گا۔ اس میں آپ کے نوڈ ایگزیکیوٹیبلز اور انہیں چلانے کے لیے درکار سیٹ اپ فائلیں شامل ہیں۔
*   log/: آپ کے نوڈز چلانے سے لاگ فائلیں شامل ہیں۔

آئیے اپنی ورک اسپیس بنائیں۔ اپنے ٹرمینل میں، چلائیں:
`ash
mkdir -p ros2_ws/src
cd ros2_ws
`

## پائتھن پیکیج بنانا

اب جب کہ ہم اپنی ورک اسپیس کے اندر ہیں، ہم اپنا پہلا پیکیج بنا سکتے ہیں۔ ہم os2 pkg create کمانڈ استعمال کریں گے جو ہم نے پہلے دیکھی ہے۔
`ash
cd src
ros2 pkg create --build-type ament_python my_first_package
`
یہ کمانڈ src کے اندر my_first_package نامی ایک نئی ڈائرکٹری بناتی ہے۔ --build-type ament_python پرچم ROS کو بتاتا ہے کہ یہ ایک پائتھن پیکیج ہوگا۔

تیار کردہ پیکیج کا یہ ڈھانچہ ہوگا:
`
my_first_package/
├── my_first_package/
│   └── __init__.py
├── resource/
│   └── my_first_package
├── test/
│   ├── test_copyright.py
│   ├── test_flake8.py
│   └── test_pep257.py
├── package.xml
├── setup.cfg
└── setup.py
`

### کلیدی فائلیں
*   package.xml: یہ پیکیج کا مینی فیسٹ ہے۔ اس میں میٹا ڈیٹا جیسے پیکیج کا نام، ورژن، مصنف، اور انحصار شامل ہیں۔ آپ کو اپنے انحصار کو شامل کرنے کے لیے اس فائل میں ترمیم **کرنی ہوگی**۔ ایک پائتھن نوڈ کے لیے، آپ کو ہمیشہ clpy کی ضرورت ہوگی۔
    `xml
    <!-- package.xml کے اندر -->
    <depend>rclpy</depend>
    `
*   setup.py: یہ ایک معیاری پائتھن سیٹ اپ فائل ہے۔ یہ colcon کو بتاتی ہے کہ آپ کے پیکیج کو کیسے انسٹال کرنا ہے اور، اہم بات یہ ہے کہ آپ کے ایگزیکیوٹیبلز کو کہاں تلاش کرنا ہے۔ آپ کو اپنے نوڈس کو رجسٹر کرنے کے لیے ایک entry_points سیکشن شامل کرنے کی ضرورت ہوگی۔
*   my_first_package/: یہ وہ ڈائرکٹری ہے جہاں آپ اپنی اصل پائتھن سورس کوڈ فائلیں ڈالیں گے۔

## ورک اسپیس بنانا

اب جب کہ ہمارے پاس ایک پیکیج ہے، ہمیں اسے بنانے کی ضرورت ہے۔ اپنی ورک اسپیس کی جڑ (os2_ws) پر جائیں اور بلڈ کمانڈ چلائیں:
`ash
cd ..  # ros2_ws کی جڑ پر واپس جائیں
colcon build
`
colcon ROS 2 کا بلڈ ٹول ہے۔ یہ آپ کی src ڈائرکٹری میں موجود تمام پیکیجز سے گزرے گا، ان کے انحصار کا پتہ لگائے گا، اور انہیں صحیح ترتیب میں بنائے گا۔

اگر تعمیر کامیاب ہوتی ہے، تو آپ کو اپنی ورک اسپیس میں نئی uild اور install ڈائرکٹریاں نظر آئیں گی۔

## ورک اسپیس کو سورس کرنا

آپ کے نئے پیکیج کے لیے ایگزیکیوٹیبلز اور وسائل اب install ڈائرکٹری میں ہیں۔ انہیں استعمال کرنے کے لیے، آپ کو اپنی ورک اسپیس کی مقامی سیٹ اپ فائل کو سورس کرنے کی ضرورت ہے۔
`ash
source install/setup.bash
`
یہ کمانڈ آپ کے ٹرمینل کو بتاتی ہے کہ آپ نے ابھی جو پیکیجز بنائے ہیں انہیں کہاں تلاش کرنا ہے۔ **آپ کو ہر نئے ٹرمینل میں یہ کرنا ہوگا جو آپ کھولتے ہیں۔**

اب، اگر آپ os2 pkg list چلاتے ہیں، تو آپ کو دستیاب پیکیجز کی فہرست میں my_first_package نظر آنا چاہیے۔

اب آپ کے پاس ایک مکمل، قابل تعمیر ROS 2 پیکیج ہے۔ اگلے سبق میں، ہم اس میں اپنا پہلا پائتھن نوڈ شامل کریں گے۔
