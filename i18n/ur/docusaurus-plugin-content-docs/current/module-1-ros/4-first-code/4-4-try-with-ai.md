# سبق 4.4: اے آئی کے ساتھ کوشش کریں

اب آپ نے دستی طور پر ایک پبلشر اور ایک سبسکرائبر لکھا ہے۔ آپ اجزاء کو سمجھتے ہیں: نوڈ کلاس، create_publisher/create_subscription کالز، کال بیکس، اور clpy.spin() لوپ۔

اب، آئیے دریافت کریں کہ ایک جنریٹو اے آئی اسسٹنٹ آپ کے کو-پائلٹ کے طور پر کیسے کام کر سکتا ہے، جو آپ کو اس کوڈ کو تیزی سے لکھنے، سمجھنے اور اس میں ترمیم کرنے میں مدد کرتا ہے۔

## اے آئی بطور "ربر ڈک"

"ربر ڈک ڈیبگنگ" ایک کلاسک پروگرامنگ تکنیک ہے جہاں آپ اپنے کوڈ کو، لائن بہ لائن، ایک ربر ڈک کو سمجھاتے ہیں۔ اسے سمجھانے کا عمل اکثر آپ پر بگ کو ظاہر کرتا ہے۔

ایک اے آئی اسسٹنٹ ایک سپر پاورڈ ربر ڈک کی طرح ہے جو واپس بات کر سکتا ہے۔

**آپ کا کام:**
1.  اپنی 	alker.py فائل کھولیں۔
2.  فائل کے پورے مواد کو کاپی کریں۔
3.  اپنے پسندیدہ بڑے لسانی ماڈل (جیسے ChatGPT, Gemini, Claude, وغیرہ) پر جائیں۔
4.  کوڈ پیسٹ کریں اور درج ذیل پرامپٹ پوچھیں۔

---
**پرامپٹ 1: کوڈ کی وضاحت**

> "آپ ایک ماہر ROS 2 ڈویلپر ہیں۔ براہ کرم مجھے یہ پائتھن کوڈ لائن بہ لائن سمجھائیں۔ میں ایک ابتدائی ہوں۔ ہر حصے کا کیا کام ہے، خاص طور پر clpy فنکشنز کی وضاحت کریں۔"

---

اے آئی کے جواب کا جائزہ لیں۔ کیا یہ آپ کی اپنی سمجھ سے مطابقت رکھتا ہے؟ کیا اس نے کوئی ایسی چیز بتائی جو آپ نے چھوڑ دی تھی؟ اے آئی موجودہ کوڈ کی تفصیلی، لائن بہ لائن وضاحت فراہم کرنے میں بہت اچھا ہے۔

## اے آئی بطور کوڈ جنریٹر

اب آئیے اے آئی سے کوڈ میں ترمیم کرنے کو کہیں۔ آئیے کہتے ہیں کہ ہم پیغام کی قسم کو ایک سادہ String سے ایک زیادہ پیچیدہ پیغام میں تبدیل کرنا چاہتے ہیں جس میں ایک کاؤنٹر اور ایک ٹائم اسٹیمپ شامل ہو۔ sensor_msgs/msg/TimeReference پیغام اس کے لیے ایک اچھا امیدوار ہے۔

---
**پرامپٹ 2: کوڈ میں ترمیم**

> "بہترین، آپ کا شکریہ۔ اب، براہ کرم 	alker.py کوڈ میں ترمیم کریں۔ میں std_msgs/msg/String پیغام کے بجائے ایک sensor_msgs/msg/TimeReference پیغام شائع کرنا چاہتا ہوں۔
>
> ٹائمر کال بیک میں، آپ کو چاہیے:
> 1. پیغام کے ٹائم اسٹیمپ کے 	ime_ref.sec حصے کو کاؤنٹر i کی موجودہ قدر پر سیٹ کریں۔
> 2. پیغام کے مواد کو اسکرین پر لاگ کرنا جاری رکھیں۔
> 3. listener.py نوڈ کے لیے بھی ایسا ہی کریں، تاکہ یہ اس نئے پیغام کی قسم کو سبسکرائب اور پارس کر سکے۔"

---

اے آئی کو آپ کو کوڈ کے دو نئے بلاکس فراہم کرنے چاہئیں، ایک ٹاکر کے لیے اور ایک لسنر کے لیے۔

**آپ کا کام:**
1.  دو نئی فائلیں بنائیں، 	alker_advanced.py اور listener_advanced.py۔
2.  اے آئی سے تیار کردہ کوڈ کو ان فائلوں میں پیسٹ کریں۔
3.  **اہم بات یہ ہے کہ آپ کو ان نئے نوڈس کو اپنی setup.py فائل میں رجسٹر کرنا ہوگا:**
    `python
    # setup.py
    'console_scripts': [
        'talker = my_first_package.talker:main',
        'listener = my_first_package.listener:main',
        'talker_advanced = my_first_package.talker_advanced:main',
        'listener_advanced = my_first_package.listener_advanced:main',
    ],
    `
4.  آپ کو اپنی package.xml میں نیا انحصار بھی شامل کرنے کی ضرورت ہے:
    `xml
    <!-- package.xml -->
    <depend>sensor_msgs</depend>
    `
5.  colcon build کے ساتھ اپنی ورک اسپیس کو دوبارہ بنائیں۔
6.  اپنے نئے نوڈس چلائیں:
    *   ٹرمینل 1: os2 run my_first_package talker_advanced
    *   ٹرمینل 2: os2 run my_first_package listener_advanced

کیا یہ کام کیا؟ اگر نہیں، تو کیا آپ اس مسئلے کو ڈیبگ کر سکتے ہیں؟ اکثر، اے آئی کوڈ کو 95% درست کر دے گا، لیکن آپ، انسانی ڈویلپر، کو حتمی 5% اصلاح اور انضمام فراہم کرنے کی ضرورت ہے۔ یہ اے آئی کی مدد سے ترقی کا جوہر ہے۔
